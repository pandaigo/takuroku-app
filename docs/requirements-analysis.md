# 卓録 要求分析（フェーズ1の正本 / 要求粒度）

> この文書は「何を・誰のために・なぜ作るか」を要求の粒度で定める。
> 画面・テーブル・技術（基本設計／詳細設計）には踏み込まない。設計の正本は別途。

## 1. 背景・課題
TRPG卓やボードゲーム会の主催者は、「その集まりが**誰と・どれだけ続いてきたか**」という関係の歴史を残せていない。
- Discord 等の連絡手段はログが流れて消える。
- 募集・日程・記録の道具がバラバラで、一つにまとまらない。
- 「続けてきたこと」を一目で見せる手段がないため、振り返りにも新メンバー勧誘にも使えない。

## 2. 対象ユーザー
- **一次対象＝卓/会の主催者（GM・キーパー・幹事）**。記録を入れる人であり、招待・勧誘の起点。エンジン。
- 二次＝掛け持ちで参加する常連（自分では記録しない）。
- 受け手＝これから入りたい人（卓のカードを見て参加を判断する）。

特に手応えが見込めるのはクトゥルフ系TRPG卓・固定メンバー＋ゲストのボドゲ会。

## 3. 利用者の本質的ニーズ
1. **続けてきた歴史が形で残る**（誰と・何回・どれだけの期間）。
2. **新しい人を誘うときに見せられる“顔”**になる（活発さ・雰囲気が伝わる1枚）。
3. **記録が一瞬で終わる**（面倒なら主催は続けない＝最優先制約）。

（補・ペルソナ会議で確認）始めた初日からカードが“顔”として機能するには、**過去の集まりも遡って残せる**ことが要る。続き物は「完走した事実」が伝わればよく、続き物の管理そのものは本筋でない。

## 4. 市場・競合の要点（調査結論の要約）
- 非AIニッチの記録サービスは少人数運営で十分な収益実例がある（ジャンルは違うが反復課金の前例あり）。
- 既存のTRPG/ボドゲ系サービスは「**遊ぶ／卓を募集する／対戦相手を探す**」を提供。
- 「**この卓の“誰と×継続×履歴”という関係グラフと、それを1枚にまとめた年鑑**」は空白。既存大手と競合せず**補完**の位置。
- マッチング/対戦の領域は世界的大手が占有済み（成功＝関係が外部で完結し離脱、という構造的弱点もある）。

## 5. プロダクト決定（要求の結論）
**TRPG/ボードゲーム会の「誰と・どれだけ続けたか」を本人入力で蓄積し、1枚の「卓年鑑カード」にする、非AIの記録サービス。**
- 集客は営業・勧誘ではなく、**カードがコミュニティで共有される（バズ）**ことで広がる前提。
- 主役は「関係の蓄積とその可視化」。遊ぶ機能・対戦機能は持たない（大手の領域・補完に徹する）。

## 6. ゴール（成功思考フレームワーク適用結果）
- 規模：**ライフスタイル規模**（個人・非AI・日本・サブスク射程で月数十万円〜が射程。大規模は狙わない）。
- 獲得：バズ（共有される1枚カード）。維持：関係とデータが積み上がるほど離れにくい。
- ゴール・俯瞰・バズ・拡散点（1枚スクショで伝わる）を満たす形であること。

## 7. 制約（要求として固定）
- **本人が入力したものだけを扱う。第三者のコンテンツを取得する経路は作らない**（法務上の絶対制約）。
- **非AI**（AI生成・AIパイプラインを核にしない）。
- **個人で保守できる規模**であること（ソロ前提）。
- **価格・課金形態は未確定。勝手に決めない**（別途協議）。

## 8. 明確に採らない案（要求段階での除外）
- 第三者コンテンツの取り込み（法務・本筋外）。
- マッチング／オンライン対戦（大手占有・成功＝離脱）。
- 施設DB・ゲームDB・シナリオDB化（正規化に寄ると大手と競合・本筋外）。
- AI機能（非AI制約）。
