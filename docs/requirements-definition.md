# 卓録 要件定義（フェーズ2の正本 / 要求粒度）

> 「何ができれば要求を満たすか」を利用者の達成基準で書く。
> 画面構成・データ構造・技術選定（基本設計／詳細設計）は**ここに書かない**。

## 1. 用語
- **卓/会**：継続して集まる単位（TRPG卓、マダミス卓、人狼会、ボドゲ会など）。
- **主催**：卓/会を管理し記録を入れる人。
- **メンバー**：その卓/会で一緒に遊ぶ人（登録作業は不要、名前だけで足りる）。
- **セッション**：1回の集まり。
- **卓年鑑カード**：その卓/会の歴史を1枚にまとめた共有用の表示。

## 2. 機能要件（利用者が達成できること）
- F1. 主催はアカウントを作りログインできる。パスワードリセット／メールアドレス変更／退会／連続失敗時の保護を含む。**メール変更は旧アドレスへ通知し、一定期間内に取消可能**。**卓削除など破壊操作前は再認証**を求める。
- F2. 主催は卓/会を作成し、その素性（名前・種別・結成日・ひとこと紹介）を持たせられる。
- F3. 主催は一緒に遊ぶ人を**名前だけ**で登録できる。本人の登録作業は不要。
- F4. 主催は1回の集まりを**必須項目だけで提出できる**。**前回出席者は既定で選択状態**。**人数が増えても手数は増えない**（10〜14人規模でも10秒目安を担保）。
- F5. 主催は記録の途中で、その場に来た新しい人をその場で追加できる。
- F6. 主催はこれまでの記録を**全件**振り返れる（古い分は分割表示）。
- F7. 主催は卓年鑑カードを生成できる（表示要素は7.6）。
- F8. 主催は卓年鑑カードを「他UIが写り込まない単独画像」または「主催が明示公開を選んだ場合に限り、ログイン無しで閲覧できる単独URL」として取り出せる。
- F9. サービス側は利用実態を残す。残すイベント：記録作成／カード閲覧／共有ボタン押下／記録の削除／公開URLの発行・失効／**公開URL経由の新規主催登録（参照元トークン付き）**／侵害通報・削除請求・対応の各イベント。
- F10. 主催は過去にあった集まりを過去日付で遡って記録できる。
- F11. 1回の集まりに複数のゲーム/シナリオが含まれてよい。**通算回数・ペースは「1集まり＝1」**として数える。
- F12. 卓年鑑カードは7.6の表示・トーン要件を満たすことを以て「活発さ・継続・雰囲気が一目で伝わる」を達成したとみなす。
- F13. 主催は**自分の全データをエクスポート**して手元に残せる。
- F14. メンバー本人からの**異議申立・削除請求の受口**がある。応答SLAは7.10。
- F15. **記録・メンバーは編集・削除できる**。メンバー削除時も過去カードの表示は壊れない。
- F16. 主催は**公開URL発行操作時、掲載される一人ひとりのメンバー名について本人同意を取得済みであること**を明示確認してから公開できる（個情法27条準拠）。
- F17. サービスは**用語のジャンル別言い換え**を持つ。種別に応じて画面文言が初心者にも分かる言い方に切り替わる（例：マダミスでは「シナリオ」、ボドゲ会では「ゲーム」）。
- F18. 新規主催は**初回ログイン→卓作成→1回の記録→カード生成→共有可能状態**まで一本道で迷わず到達できる。途中で過去遡及記録の価値が自然に提示される。
- F19. カード共有時、受け手に**「自分も卓を作れる」と気づける導線**が同梱される。
- F20. 公開URLが外部SNSに貼付された時、**プレビュー（画像と短文）で要点（卓名・通算・期間・称号）が伝わる**。プレビュー画像は他UIが写り込まない。
- F21. **侵害情報の通報窓口**がカード公開URLからアクセス可能。処理フローは7.12。

## 3. 非機能要件（要求レベル）
- N1. 反応が速く軽い。記録は思い立った直後に完了できる。画面が固まったように見えない。
- N2a. **縦画面の片手親指操作**で全機能に到達できる。
- N2b. 文字サイズ・タップ領域はアクセシビリティ基準の下限を確保する。
- N3. 他人の卓・記録は一切見えない。例外：主催が明示公開を選んだ卓年鑑カードのみ公開URL経由でログイン無しで閲覧される。
- N4. 日本語UI（自然な日本語）。**専門用語は種別ごとの言い換え（F17）で配慮**。
- N5. 非AIで成立する。
- N6. 公開Webで誰でも到達できる。
- N7. 個人で落とさず保守できる規模・複雑度に収める。
- N8. 日本市場を対象とする（海外展開はスコープ外）。
- N9. **アクセシビリティ達成**：JIS X 8341-3 相当を目標とし、視覚・運動・認知・前庭の多様性に配慮する（達成項目は7.8）。
- N10. **共有時の品格**：カードは外部SNSに貼付された状態で「卓録のカード」と識別でき、広告・煽り画像と誤認されない品位を保つ。

## 4. データ・プライバシー要件
- D1. 本人が入力したものだけを保持する。
- D2. 第三者のコンテンツを取得する経路を作らない（絶対制約）。
- D3. 認可は N3 に従う。
- D4. 保持する個人データ項目（メールアドレス／表示名／卓・メンバー・セッション・カード）を要件として明示する。
- D5. メンバーは「実名でなく卓内呼称」を推奨。主催は事前同意取得義務を規約レベルで合意する。**公開URL発行時の同意確認はF16**。
- D6. 退会時：主催の個人特定情報は削除。配下の卓・メンバー・セッション・カードを同時に削除する。
- D7. 卓削除時：配下のメンバー・セッション・カードを連鎖削除する。
- D8. メンバー本人からの異議申立・削除請求を受け付け、**応答SLAは7.10**に従う（通常10営業日、緊急3営業日）。
- D9. 共有カードの公開URLは推測不能なトークンを持ち、主催がいつでも失効できる。
- D10. **編集履歴は卓データ側に残さない**（自己申告の歴史）。**対応事務記録は別系統で保持**（D20）。
- D11. 13歳未満は利用不可。13〜17歳は保護者同意を前提とする（取得方法はD22）。
- D12. 自由入力欄（ひとこと／名場面／紹介）には「第三者を識別する情報を書かない」ことをガイドラインとして提示する。
- D13. **PP/利用規約の整備**：利用目的を特定し、登録前および各取得時点で利用者が確認できる形で提示する。
- D14. **安全管理措置の方針**：技術的・組織的・人的・物理的安全管理措置の方針を定め、PP概要に記載する。
- D15. **漏えい等報告・本人通知**：個情委への速報・確報の報告、本人通知、報告窓口責任者を要件として持つ。
- D16. **越境転送の同意**：海外クラウド事業者を利用する場合、所在国と移転に関する情報をPPに明記し、必要に応じて同意を取得する。
- D17. **権利侵害通報窓口**：公開URLには侵害情報通報窓口を常時掲示し、応答SLAは7.10に従う。
- D18. **退会・公開停止時**：URLを即時失効し、外部キャッシュ削除を試みる運用を持つ。
- D19. **ログ保持期間と利用目的**：F9の各ログについて保持期間と利用目的をPPに記載する。
- D20. **対応事務記録の分離保持**：削除請求対応・通報対応の事務的記録は卓データと別系統で保持する（説明責任の証跡）。
- D21. **規約変更等の取り決め**：規約変更時の事前周知方法、準拠法（日本法）、合意管轄（事業者所在地の地裁）、事業者の損害賠償上限の予定を規約に置く。
- D22. **年齢確認・保護者同意**：登録時に生年月日を取得。13歳未満は不可。13〜17歳は保護者の電子的同意（保護者メール宛確認等）を取得する。
- D23. **将来課金時の準備（予備宣言）**：特商法11条表記、サブスク自動更新事前通知、最終確認画面、解約導線が登録より重くないこと、クーリングオフ非適用の明示、未成年契約取消対応を満たす。

## 5. スコープ外（やらないことを明示）
- 招待リンク／相手への自動共有／相互可視（次段）。
- 自動通知・年次自動配信。
- マッチング／オンライン対戦。
- ゲーム・シナリオ・施設のデータベース化。
- AI機能。
- 課金の実装（価格未確定）。
- 続き物（キャンペーン／シリーズ）の独立した束ね・管理機能（次段）。
- 同卓内「誰と誰が何回一緒だったか」共起グラフのカード表示（次段）。
- 卓データ側の編集履歴・改ざん不能監査ログ（運用記録は D20 で別系統）。
- 海外展開。

## 6. 成功の定義（受入の質的基準）
- S1. 実際の主催が**催促されなくても記録を続ける**。
- S2. 主催が**卓年鑑カードを共有・勧誘に使う**。
- S3. **共有を起点に別の主催が新規利用に至る経路**が観測される（F9拡張の参照元トークンで観測）。
- S4. 卓あたりの**継続が3ヶ月以上で複数事例**観測される。

> 数値ゲートは規律として持たない方針。

## 7. 機能要件 詳細仕様

### 7.1 ログイン・新規登録・アカウント管理
- **入力**：メールアドレス（必須・形式）／パスワード（必須・8文字以上）
- **操作**：ログイン／新規登録
- **エラー文**：認証失敗／パスワード長違反／確認メール送付の旨
- **アカウント管理**：
  - パスワードリセット（登録メールに再設定リンク）
  - メール変更（本人確認後）。**旧アドレスに通知。一定期間内に取消可**。
  - 退会（本人操作）。挙動はD6。
  - 連続失敗時の保護（一定回数で一時制限）。
  - **破壊操作前の再認証**（卓削除等）。

### 7.2 卓/会の作成
- **入力**：卓名（必須・上限50）／種別（必須・5択）／システム名（任意）／結成日（任意）／ひとこと紹介（任意・上限200）
- **種別の選択肢**：`TRPG卓` / `マーダーミステリー卓` / `人狼会` / `ボドゲ会` / `その他`、既定「TRPG卓」
- **公開化操作**：F16/D5に従い、メンバー名公開の同意取得済みを明示確認してから公開できる。
- **エラー文**：卓名なし／作成失敗

### 7.3 メンバー登録
- **入力**：名前（必須）／役割（任意）
- **同名**：許可（誤入力修正は F15）
- **本人登録**：不要

### 7.4 セッション記録
- **入力**：日付（必須・既定今日・過去日付OK）／タイトル（必須・複数併記可）／来た人（複数選択・**前回出席者既定**・0人OK・**大人数モード（10人以上で一括選択挙動を持つ）**）／新しい人（自由入力・カンマ等区切り）／結果（任意・なし/クリア/進行中/全滅TPK）／ひとこと
- **振る舞い**：必須項目のみで送信可。**人数増でも10秒目安を担保**。1集まり＝1カウント。二重送信防止。不正日付は安全に拒否。日付上下限なし。前後空白切詰め、絵文字・多言語可。

### 7.5 履歴一覧
- 表示項目：日付／タイトル／結果バッジ／来た人／ひとこと
- 並び順：日付の新しい順
- **全件閲覧可能・古い分は分割表示**

### 7.6 卓年鑑カード
**表示項目**
- 卓名／種別ラベル／システム名（あれば）
- 通算セッション数／結成からの期間／ペース（月◯.◯回）
- 完走シナリオ数（0時は非表示）
- メンバー：参加回数順「名前（回数）」中黒連結。**人数多時は上限と省略表記**。
- 名場面：直近3件。**本文と区別される引用形式**（誰の言葉・どの回か特定可）。
- 称号：通算回数＋続けた期間（ペース）加味で決定論。**卓の継続事実に由来する語彙のみ、煽り・優劣評価語禁止**。具体閾値は設計で。
- ひとこと紹介＋「一緒に遊ぶ人募集中」（あれば併記）

**ジャンル別の重み付け**
- マダミス：1回完結文化に配慮。「完走シナリオ数」「異なる共演者の延べ人数」を主指標として扱う。
- 人狼会：大人数前提。メンバー欄が視覚的に潰れない上限・省略を持つ。
- TRPG卓：継続期間・ペースを主指標。
- ボドゲ会：通算とメンバー積み上げを主指標。

**ジャンル別トーン**
- 既定は**種別から導かれるトーン**。主催は「種別準拠 / 統一」を選べる。

**感情・トーン**
- 「**継続・楽しさ・絆**」のうち最低1つを、数値以外の要素（引用・称号・人名の連なり等）で伝達する。
- カードは**統計ダッシュボードでなく年鑑/記念誌**の佇まいを保つ。人名と引用が中心、数値は脇役。

**データが少ないとき**
- セッション 0：通算 0／期間「1ヶ月」（下限）／ペース「月 0」
- **下限を満たさない場合は「これから／門出」相当の表現**で気持ちを冷やさない
- メンバー 0：「まだいません」表示
- 名場面 0：ブロック非表示

**画像化（要件）**
- 他UIが写り込まない単独画像として書き出せる

**外部共有のための公開URL（採用）**
- 主催の明示公開時のみ、ログイン無しで閲覧可能
- 推測不能なトークン／いつでも失効可
- **公開化操作時の同意確認**（F16/D5）
- **テキスト代替表現**：公開URL側にスクリーンリーダー等でも内容が読めるテキスト構造（卓名・通算・期間・ペース・メンバー名と回数・称号・名場面）
- **OGP/共有プレビュー**：F20に従い、要点が画像と短文で伝わる
- **受け手向け導線**：F19に従い、「自分も卓を作れる」気づきを与える
- **検索エンジンインデックス**：主催が公開範囲を選べる
- **侵害通報窓口**：F21/D17/7.12 に従い、画面から到達可能

**操作**
- 「このカードを共有・勧誘に使った」ボタン → 確認文表示
- 「卓にもどる」リンク

### 7.7 認可・プライバシー
- 自分の卓・記録は自分（および将来の権限者）だけが見える
- 例外：主催が明示公開を選んだ卓年鑑カードのみ、公開URL経由で閲覧可能
- 第三者コンテンツ取り込み経路は無い
- メンバー本人からの異議申立・削除請求の受口がある（D8／7.10／7.12）

### 7.8 表示・操作・アクセシビリティの共通要件
- **色のみで情報を伝達しない**（色＋形状/テキスト併用）
- **主要操作はキーボード・支援技術でも完結**できる
- **十分なコントラストと文字サイズ**を確保する
- 文字サイズ拡大時もカードの主要項目が欠落しない
- 動き・アニメーションはOSの「視差効果を減らす」設定に従う
- **複雑なジェスチャを必須にしない**（単一タップの代替手段あり）
- エラー文・操作指示は専門用語に頼らず、原因と次の一手が短く分かる
- スマホ縦画面の片手親指操作で完結（N2a）
- 各サブ画面に「もどる」リンク
- 表示遅延時に画面が固まったように見えない

### 7.9 注記
同卓内「誰と誰が何回一緒だったか」の共起グラフ表示は本フェーズで作らない（次段候補）。

### 7.10 運用・サポート要件
- **問い合わせ・申立の受付経路**：サービス内に明示された経路（メールまたはフォーム）
- **応答SLA（目安）**：
  - 受領通知 **5営業日以内**
  - 一次回答 **10営業日以内**
  - 削除請求（通常） **10営業日以内**
  - 削除請求（緊急＝晒し・なりすまし等） **3営業日以内**
  - 障害第1報 **検知後24時間以内**
- **長期不在告知**：7日以上の不在は事前告知
- **データバックアップ**：日次でバックアップ、消失時は事象と影響範囲を告知
- **処理上限**：個別対応は月次想定件数を上限とし、超過時は遅延を告知
- **サービス終了告知**：**60日前**まで／期間中はエクスポート可
- **不利益変更（料金・規約）**：**30日前**までに登録メールおよびサービス内で告知
- **アンチパターン禁止**：退会・公開URL失効・データエクスポートは登録時と同等の手数で完了できる

### 7.11 オンボーディング要件
- 新規主催は**初回ログイン→卓作成→1回の記録→カード生成→共有可能状態**まで一本道で迷わず到達できる
- 途中で**過去遡及記録の価値**（今日からカードが"顔"になる）が自然に提示される
- **種別選択直後に用語ガイド**が提示される（種別ごとの言い換え＝F17）
- **卓0件**の主催に対し、最初の卓作成と過去遡及を促す案内が画面上に提示される
- **リテンション（プル型）**：主催が再訪時に「前回からの差分（経過日数・直近の名場面）」を即座に確認できる

### 7.12 侵害・ハラスメント救済フロー
- **発見可能性**：被害者向け案内ページが検索可能。**カード公開URLから申立窓口に到達できる**。
- **申立**：ログイン不要で、対象URL／自分の表記／被害種別／連絡先で申立可能。第三者代理申立も受ける。
- **暫定措置**：本人性が一定程度確からしい申立を受けた時点で、対象記載の公開閲覧を**暫定停止**できる（主催のデータは消さない）。
- **主催側手続**：主催に通知し、反論・修正の機会を与える。期限内無応答なら公開停止継続。
- **恒久対応**：申立成立で当該メンバー名・関連自由入力欄の**公開からの除去**（参加履歴の内部保持はD6/D7範囲）。
- **再発抑止**：重大違反の累積でアカウント側に措置できる（要件として「累積を見て措置できる」）。
- **濫用検知**：通報の重複・濫用検知。**虚偽通報の累積でレピュテーション低下**、自動非公開ではなく一次審査。
- **事務記録**：申立・判断・措置・通知を運用記録として残す（D20で別系統）。

### 7.13 なりすまし対策
- 同一/酷似卓名の検知、「公式」「運営」等を騙る表現の禁止、被詐称者の申立で表示停止
- 過去に重大違反で停止された主体の再登録抑止（同一性完全特定不能でも可）
